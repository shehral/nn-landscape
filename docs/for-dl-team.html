Patterns this build

  • Five agentic-platform and RAG orchestration repos (Dify, RAGFlow, BiSheng, UltraRAG, GenKit) trended in the same build with no confirmed Nanonets extraction connector on any — the distribution gap spans the full orchestration tier simultaneously.
  • The open-source document-processing stack (MinerU, OCRmyPDF, Tesseract, paperless-ngx) trended alongside VLM-native extraction tools in the same cycle, indicating the market has not converged on a single extraction architecture — legacy text-layer OCR and LLM-native JSON extraction remain parallel investment areas.
  • ms-swift and Ollama co-trended with featured models Qwen3-VL, InternVL3.5, and GLM-5.1, indicating that the fine-tuning and local-serving pipeline for IDP Leaderboard-competitive models is now accessible with commodity tooling — the path from model selection to document-task fine-tune no longer requires specialized infrastructure.
  • Two VLM serving frameworks (SGLang, TokenSpeed) and Qualcomm's edge-deployment SDK trended in the same build, reflecting multi-directional cost-and-latency pressure: cloud inference cost reduction and edge inference enablement are both active fronts, squeezing the API economics for mid-scale document-AI providers.
  • PixelRAG (pixel-native visual search) and Headroom (pre-LLM context compression) both trended as architectural alternatives to text-extraction-first pipelines — one bypasses extraction via visual embedding, the other reduces token volume before the model stage. Neither is production-proven on structured documents, but both represent documented alternatives to structured extraction APIs.
