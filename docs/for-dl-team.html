Patterns this build

  • The complete open-source document AI pipeline spans every layer simultaneously: MinerU and opendataloader-pdf for extraction, PaddleOCR and Unstructured as processing alternatives, OCRmyPDF as a Tesseract-backed baseline, and Dify, RAGFlow, and UltraRAG for orchestration — all trending in a single build cycle.
  • The GRPO fine-tuning-to-evaluation supply chain for IDP Leaderboard competitors is complete in a single build cycle: ms-swift covers GRPO training for every competitor family (Qwen3-VL, Qwen3-Omni, InternVL3.5, GLM-5.1, GLM4.5v), evalscope and lmms-eval cover benchmark evaluation, and LLaVA-OneVision-1.5 provides an additional training base.
  • OCR-3 is absent from both lmms-eval and evalscope model registries while every named IDP Leaderboard competitor (Qwen3-VL, InternVL3.5, GLM4.5v, GLM-OCR) is registered; competitors can publish benchmark comparisons via either framework that structurally exclude OCR-3 as a baseline today.
  • Spatial evidence anchoring — verifying extracted content against a verifiable location in the source media — emerges as a cross-domain pattern across three independent repositories this cycle (NVIDIA VSS Blueprint, video-evaluator, ArtSeek); applied to document tables, rows without a visual anchor in the source document image are candidate phantom rows.
  • Three orchestration platforms (Dify, RAGFlow, UltraRAG) trend simultaneously without confirmed Nanonets Agentic Data Extraction connectors; all three bundle open-source extraction defaults (Unstructured, PaddleOCR) that also trend this cycle, completing a distribution-gap feedback loop.
