Patterns this build

  • The full open-source document AI pipeline is represented in a single trending cycle: extraction (MinerU, opendataloader-pdf, OCRmyPDF), evaluation (lmms-eval, evalscope), fine-tuning (ms-swift, LLaVA-OneVision-1.5), and orchestration (Dify, RAGFlow). Each layer independently reduces the friction of substituting Nanonets with open-source alternatives.
  • The GRPO fine-tuning supply chain for IDP Leaderboard-proximate models is complete: ms-swift provides GRPO for Qwen3-VL, InternVL3.5, and GLM-5.1; LLaVA-OneVision-1.5 provides the vision backbone; lmms-eval and evalscope close the evaluation loop. An external lab can run a credible IDP Leaderboard challenge build without any proprietary infrastructure.
  • Three retrieval-based VLM hallucination mitigation paradigms appeared in the same build: self-reflective RAG, knowledge-infused multimodal retrieval, and late-interaction retrieval (ArtSeek). The concurrent appearance suggests a research moment rather than a single-lab trend, with each addressing different aspects of VLM hallucination via retrieval rather than mechanistic interpretability.
  • Distribution channel fragmentation is expanding: Dify, RAGFlow, UltraRAG, genkit, and PySpur are all trending without confirmed Nanonets connectors. These five platforms collectively represent the developer interface tier above the extraction layer; defaults set there are sticky and compound across successive build cycles.
  • Local model runners are now distributing IDP Leaderboard-proximate competitive models as default installs: Ollama's description names GLM-5, Kimi-K2.5, Qwen, and DeepSeek. Combined with Open-WebUI as the interface layer, a developer benchmarking OCR-3 alternatives can do so on-device with zero API spend.
