Patterns this build

  • Open-source document-to-markdown extraction has reached parity with managed-API positioning: MinerU, unstract, and RAGFlow all describe themselves in terms previously exclusive to commercial extraction APIs (agentic workflows, LLM-ready JSON, ETL pipelines), compressing the differentiation argument for managed services to accuracy, support, and compliance rather than capability.
  • IDP Leaderboard competitors are now locally runnable at zero marginal cost: GLM-5.1, Qwen, and Kimi-K2.6 in Ollama's featured list means the top-ranked open-weight models on OmniDocBench and DocVQA are accessible to enterprise buyers without API fees, narrowing the cost-per-page advantage of cloud extraction APIs.
  • Multimodal serving is converging on omni-modal frames: both SGLang and vllm-omni trend simultaneously, each supporting combined text + vision inference in a single serving process. This signals the serving-infrastructure community moving from modality-specific inference stacks to unified endpoints.
  • Document management and RAG platforms increasingly embed their own parsing layers: paperless-ngx, RAGFlow, and LlamaIndex all include native document ingestion, reducing the number of pipeline steps that route through a third-party extraction API. Each embedded parser is a potential non-consumption event for API-based extraction.
  • VLM evaluation tooling is bifurcating into academic harnesses (lmms-eval, MTEB) and pixel-native retrieval frameworks (PixelRAG), reflecting two distinct measurement paradigms: file-based extraction accuracy versus visual search over rendered pixels. OCR-3 is registered in neither harness, meaning third-party reports from either track exclude it structurally.
