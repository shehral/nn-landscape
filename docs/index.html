Patterns this build

  • The open-source document AI stack is now end-to-end within a single trending cycle: extraction (Unstructured, opendataloader-pdf), orchestration (Dify, RAGFlow), evaluation (lmms-eval, evalscope), and fine-tuning (ms-swift) are all present simultaneously, indicating that an all-open-source replacement for a commercial document-AI service is now technically achievable for technically-capable teams.
  • Both major third-party VLM evaluation toolkits (lmms-eval and evalscope) are trending without OCR-3 in their model registries, creating a structural gap: competitors publishing benchmark results through either framework produce head-to-head comparisons that omit the flagship model by default.
  • GRPO fine-tuning of IDP Leaderboard-comparable document VLMs (Qwen3-VL, GLM-5.1, InternVL3.5) is now accessible via ms-swift, indicating that near-term leaderboard position improvements from external labs are more likely to arrive via accessible fine-tuning than new model architectures.
  • Multiple agentic orchestration platforms (Dify, RAGFlow, UltraRAG, LangChain) are trending concurrently, suggesting the orchestration layer is commoditizing faster than the extraction layer; competitive risk shifts toward which extraction service becomes the default inside these platforms rather than standalone extraction API adoption.
  • VLM fine-tuning infrastructure (LLaVA-OneVision-1.5, TinyLLaVA_Factory, ms-swift) is trending together, signaling an acceleration in the supply of fine-tunable base models; this increases the expected rate of new document-specific VLM IDP Leaderboard entries in the next 1-3 build cycles.
