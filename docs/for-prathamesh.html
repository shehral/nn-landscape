Patterns this build

  • Three open-weight document extraction tools headlined with 'PDF to structured data for AI' messaging in the same build cycle: PaddleOCR (96.33 OmniDocBench v1.6 #1), MinerU (Shanghai AI Lab), and Chandra OCR 2 (Datalab). All three are open-source and free, and all three use positioning language nearly identical to Nanonets Agentic Data Extraction.
  • The training-to-distribution loop for competitive document VLMs is now fully self-contained at consumer scale: ms-swift enables GRPO/DPO/SFT fine-tuning of Qwen3-VL and GLM-5.1, while Ollama distributes the same model families locally. A team outside a frontier lab can now produce and deploy a custom document extraction VLM without cloud infrastructure.
  • Enterprise document workflow orchestrators trended as a cluster: RAGFlow, BiSheng, Dify, LangChain, and FlowiseAI all appeared in the same build cycle. This is the orchestration layer above document extraction — these platforms set default parser behavior and determine which extraction API each enterprise workflow calls.
  • On-device VLM inference tooling expanded simultaneously: Qualcomm GenieX (NPU/GPU/CPU) and RunanywhereAI (local production toolkit) both trended, adding to prior-build appearances of Ollama's local model distribution and LightOn OCR-2 on Qualcomm hardware. The enablement stack for on-device document VLMs is maturing faster than the question of customer demand for it.
  • VLM faithfulness and evaluation tooling co-trended: crossroute-audit (explanation-faithfulness auditing) and evalscope (benchmark evaluation framework covering DocVQA and ChartQA). Together they provide coverage of two distinct evaluation dimensions — mechanistic faithfulness and task accuracy — that are separately active in the document VLM research community.
