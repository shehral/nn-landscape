Patterns this build

  • The GRPO document-VLM supply chain is now functionally complete in a single build cycle: ms-swift provides GRPO fine-tuning for every primary IDP Leaderboard competitor family, lmms-eval provides evaluation, and Ollama provides local inference. An external lab can fine-tune, evaluate, and deploy an IDP Leaderboard-competitive document VLM without cloud API access.
  • Three independent behavioral grounding paradigms co-trend: video-evaluator (spatial evidence anchoring), artseek (late-interaction retrieval), and crossroute-audit (explanation faithfulness). Each verifies VLM output without model internals access, providing a lightweight complement to mechanistic interpretability for phantom-row detection.
  • The open-source document-extraction stack covers every pipeline tier in a single cycle: PaddleOCR and MinerU at OCR/parsing, Unstructured and opendataloader-pdf at structured extraction, RAGFlow and Dify at orchestration. A complete document-to-RAG pipeline is assemblable from OSS components without a managed API.
  • Distribution consolidates around Ollama and genkit: Ollama lists every IDP Leaderboard competitor for local inference; genkit registers Anthropic, OpenAI, and Gemini as connectors for AI-app development. Both distribute competitor capabilities at zero marginal cost with no confirmed Nanonets connector in either platform.
  • DeepSeek expands its footprint simultaneously across architecture research and fine-tuning: Janus-Pro (unified multimodal understanding-and-generation) is now available via community PyTorch reproduction, while ms-swift GRPO explicitly covers DeepSeek-V4. DeepSeek now has an architecture-research presence alongside its IDP Leaderboard entry (DeepSeek-OCR 2, 91.09% OmniDocBench v1.5).
