Patterns this build

  • Open-source PDF-to-LLM extraction toolkits converge on the same markdown/JSON output interface. MinerU, opendataloader-pdf, and PaddleOCR are backed by distinct lab and community efforts yet expose functionally equivalent extraction contracts, signaling commoditization pressure on the API surface that managed extraction vendors occupy.
  • GRPO fine-tuning is reaching multimodal VLMs through accessible open-source tooling. ms-swift and xtuner both now support GRPO for Qwen3-VL, InternVL3.5, and GLM-5.1, making RL-based capability gains previously confined to text models available for document VLM fine-tuning without specialized infrastructure.
  • Multimodal evaluation is consolidating around unified open toolkits. lmms-eval now covers DocVQA, ChartQA, and IDP-adjacent tasks in a single harness alongside video and audio; as this becomes the default evaluation substrate, third-party benchmarks against OCR-3 become easier to publish without IDP Leaderboard coordination.
  • Above-extraction agentic orchestrators are the fastest-growing open-source category in the document-AI stack. Dify and RAGFlow both position as full-stack document workflow platforms that embed parsing as a sub-component; the trend is toward owning the orchestration layer and treating extraction as a commodity.
  • Omni-modality model support is propagating downstream to serving infrastructure. vllm-omni extends vLLM to models that handle text, image, and audio in a single inference pass; as frontier labs ship omni models with document-parsing capability, self-hosted deployments of such models become a low-cost substitute for managed extraction APIs.
