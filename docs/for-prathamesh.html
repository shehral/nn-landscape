Patterns this build

  • Complete GRPO fine-tuning supply chain for IDP Leaderboard competitor model families assembled in a single cycle. ms-swift (GRPO training for Qwen3.6/GLM-5.1/InternVL3.5), evalscope (VLM benchmarking), and LLaVA-OneVision-1.5 (fine-tune base) all trend together; the barrier to a competitive IDP Leaderboard submission has dropped from engineering effort to execution cost.
  • Open-source document extraction tools converge on identical positioning language to Nanonets Agentic Data Extraction. MinerU ('LLM-ready markdown/JSON for Agentic workflows'), opendataloader-pdf ('PDF Parser for AI-ready data'), and Skill_Seekers (PDFs-to-Claude-skills) each frame their output as LLM-pipeline-ready. The competitive risk is messaging convergence, not only feature parity.
  • Three serving and training infrastructure frameworks applicable at OCR-3's scale (35B MoE) trend simultaneously. SGLang (multimodal serving with constrained decoding), NeMo Automodel (distributed VLM training), and XTuner (ultra-large MoE training) are all publicly available; the end-to-end infrastructure to train, fine-tune, and serve competitive document VLMs at OCR-3's scale now exists in open tooling.
  • Three distinct VLM hallucination mitigation paradigms appear in a single build: serving-layer constrained decoding (SGLang), spatially-grounded visual anchoring (video-evaluator), and retrieval-side context augmentation (Knowledge-Infused Multimodal Retrieval). These approaches target different pipeline stages and have not been compared against each other or against the team's mechanistic interpretability work on a common phantom-row benchmark.
  • Above-extraction-layer agent platforms are converging with the extraction-layer competitive set in the same build cycle. Dify, LangChain, RAGFlow, and Ollama each trend with document-workflow or agent-platform positioning; none has a confirmed Nanonets OCR-3 connector. LangChain's repositioning as 'the agent engineering platform' makes the integration-partner framing in nanonets_context.md ambiguous.
