Patterns this build

  • Open-source document-AI stack is now full-coverage across all tiers: MinerU and opendataloader-pdf at the extraction layer, RAGFlow and Dify at orchestration, and OCRmyPDF and paperless-ngx at the archiving layer all appear in a single build cycle. No commercial component is required to assemble a complete document pipeline.
  • Benchmark-framework exclusion gap widens: lmms-eval and evalscope both trend with competitive VLMs (Qwen3-VL, GLM-5.1, InternVL3.5) in their model registries while OCR-3 is absent; simultaneous ms-swift availability means competitors can fine-tune and benchmark against OCR-3 substitutes without OCR-3 appearing as the reference baseline.
  • GRPO fine-tuning for document VLMs is becoming commoditized: ms-swift now provides a unified GRPO pipeline for Qwen3-VL, GLM-5.1, and InternVL3.5, while LLaVA-OneVision-1.5 provides a complementary base for fine-tuning; both toolchains reduce the GPU budget required to produce a credible IDP Leaderboard submission.
  • Retrieval-based hallucination mitigation recurs in the same cycle as an independent paradigm: self-reflective-rag and Knowledge-Infused Multimodal Retrieval both address phantom-row and structural hallucination via retrieval augmentation rather than mechanistic interpretability, continuing a pattern from prior builds without a confirmed team stance on whether this line is inside the research scope.
  • Distribution-channel gaps accumulate across orchestration frameworks: Genkit (Google-backed), UltraRAG (MCP-native), Dify, and RAGFlow all trend without confirmed Nanonets connectors; each platform has pre-integrated alternatives, meaning the gap is not technical but a connector-submission omission that grows with each build cycle.
