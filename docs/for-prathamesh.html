Patterns this build

  • The open-source document AI stack reached full coverage in a single cycle: extraction (MinerU, PaddleOCR, opendataloader-pdf, OCRmyPDF, Tesseract), RAG+orchestration (RAGFlow), and evaluation (lmms-eval, evalscope) all trended simultaneously without a commercial dependency in the loop.
  • Evaluation frameworks are ahead of model registration: lmms-eval and evalscope both trended with document-AI benchmark coverage (DocVQA, ChartQA, OmniDocBench-adjacent tasks) while OCR-3 appears in neither model registry, creating a structural exclusion risk if competitors publish head-to-head results first.
  • MCP-native orchestration platforms are consolidating around a shared integration surface: UltraRAG, Dify, and RAGFlow trended in the same cycle as production-ready document workflow platforms, all without confirmed Nanonets connectors, suggesting a distribution channel gap where Nanonets is currently absent.
  • VLM fine-tuning infrastructure is lowering the IDP Leaderboard submission barrier: ms-swift (GRPO for Qwen3-VL/InternVL3.5/GLM-5.1), LLaVA-OneVision-1.5, and Ollama (now supporting Kimi-K2.6 and GLM-5.1 locally) trended together, creating a complete low-cost pipeline from fine-tuning to local serving for potential leaderboard challengers.
  • Serving infrastructure decisions remain open across multiple cycles: SGLang, RunanywhereAI, and ms-swift's large-model fine-tuning support all trended in the same build, yet no team decision on OCR-3 serving infrastructure has been recorded across six or more consecutive builds in which these items have appeared.
